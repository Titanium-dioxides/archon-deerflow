## 激活方式

在 DeerFlow 的对话窗口中直接输入数学命题即可。Skill 会根据触发词自动激活。

**触发关键词**（满足任意一个即可）：

| 关键词 | 示例 |
|--------|------|
| "证明" | "请证明根号2是无理数" |
| "求证" | "求证：素数有无穷多个" |
| "命题" | "检查以下命题是否成立：..." |
| "定理" | "证明费马小定理" |
| "prove" | "Prove that the square root of 2 is irrational" |
| 含有 `<proof>` 标签 | 已有证明内容需要验证 |

## 基本用法

### 1. 标准证明请求

```
请证明：√2 是无理数。
```

流程：生成 → 验证 → 通过 → 输出最终证明。

### 2. 验证已有的证明

```
命题：n² 为偶数则 n 为偶数。
我的证明：假设 n 是奇数，则 n = 2k+1，n² = (2k+1)² = 4k²+4k+1 = 2(2k²+2k)+1 是奇数，与假设矛盾，因此 n 是偶数。
请验证这个证明是否正确。
```

流程：跳过生成，直接进入验证模式输出 JSON。

### 3. 带定理搜索的证明

```
证明：素数定理（即 π(x) ~ x/ln(x)）。
```

流程：**Step 0 [检索]** 自动搜索 leansearch.net 获取相关引理 → **Step 1 [生成]** → **Step 2 [验证]**。

### 4. 证明难度较大时（期望多次修复循环）

```
证明：所有奇数的平方模8余1。
```

预期：生成 → 验证 → 修复 → 验证 → 通过（2 轮内完成）。

## 工作流示意图

```
你输入命题
    │
    ▼
┌─────────────────────────────────────┐
│  0. [检索模式]                       │
│  python search.py "命题关键词"       │  ← 可选，自动触发
│  → [CONTEXT] 注入下一步              │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  1. [生成模式]                       │
│  读取 prompts/generator.md           │
│  → 输出 <proof>...</proof>          │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  2. [验证模式]                       │
│  读取 prompts/verifier.md            │
│  → 输出 JSON                        │
└─────────────────────────────────────┘
    │
    ├── verdict="correct" ──→ ✅ 输出最终证明
    │
    └── verdict="wrong"  ──→ 修复（≤ 2 次）
              │
              └── 第 3 次仍 wrong ──→ ❌ 输出失败报告

```

## 用例模板

### 证明真命题

```
> 请证明：√2 是无理数。

<proof>
...标准反证法...
</proof>

{ "verdict": "correct" }

✅ 最终证明：
……（格式化后输出）
```

### 否定伪命题

```
> 请证明：所有素数都是奇数。

<proof>
结论：该命题不成立。反例：p=2。
</proof>

{ "verdict": "correct" }

✅ 证明完成：该命题错误，反例已给出。
```

### 修复循环触发

```
> 证明：两个无理数之和必为无理数。

<proof>（第一次生成，有漏洞）</proof>

→ 验证输出：
{
  "critical_errors": [
    {"location": "第2段", "issue": "反例：√2 和 (-√2) 都是无理数但和为 0 是有理数"}
  ],
  "verdict": "wrong",
  "repair_hints": "需要修正命题为'两个无理数之和不一定是无理数'..."
}

→ 修复重生成：
<proof>（第二次生成，修正后）</proof>

→ 验证输出：{ "verdict": "correct" }

✅ 最终证明：原命题错误，反例 √2 + (-√2) = 0。
```

### 未解难题（触发 3 次上限）

```
> 证明：哥德巴赫猜想。

← 第 1 次 → { "verdict": "wrong", ... }
← 第 2 次 → { "verdict": "wrong", ... }
← 第 3 次 → { "verdict": "wrong", ... }

❌ 失败报告：
尝试 3 次均未通过验证。最后一次草稿及审稿反馈如下：
...
```

## 故障排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 明明错的说对的 | 模型验证时放水 | 回到对话，重新发送"你再检查一遍，不要放过任何漏洞" |
| JSON 解析失败 | 模型在 JSON 前后加了废话 | 提醒模型："输出必须是纯 JSON，不要加任何额外文字" |
| 循环到后期忘记命题 | 上下文太长 | 在你下一次输入中重申原始命题 |
| 速度太慢 | 证明太长 | 提示模型"使用简写符号，保持证明在 500 字以内" |
| 搜索无结果 | leansearch.net 不可达 | 不影响，模型会用内置知识继续生成 |
